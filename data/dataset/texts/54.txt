WEB Стационар: Приемный покой

h1. Приемный покой

h2. Создание МКСБ

Врач приёмного отделения
Для создания нового стационарного случая нажмите на кнопку {{b5555a36-bb6c-41fb-9602-fd0403940635.png}}  в журнале пациентов и выберите тип истории: МКСБ, История родов или История развития новорожденного.

{{3a21ff71-b7d0-4574-93d2-9458cf9f2cb8.png}}
 
Откроется форма для поиска пациента. Для поиска пациента введите критерии поиска, затем нажмите на кнопку «Найти». Отражаются данные из источника МКАБ и МКСБ.
Результат поиска представлен в виде таблицы, отображается только 5 совпадений.

{{77353a8e-2dd4-4cb3-b99d-7a4e7ddae4d9.png}}
 
Если в результате поиска не найдено ни одной записи, появится сообщение, что пациенты не найдены. Будет предложено создать новую карту.
{{f78be37d-f449-4605-b483-cfed2b409d6f.png}} 
После нажатия кнопки «Создать новую карту» либо выбора нужного пациента будет открыта форма создания МКСБ. Слева располагаются разделы карты. С помощью левой панели можно быстро переходить в нужный раздел.
Для удобства работы с картой она разделена на блоки.
{{e6510796-57bf-4784-90df-9dde1ba4bc19.png}} 

Поля помеченные символом «h1. » обязательны к заполнению, если поля не заполнены, то они будут выделены красным цветом.

h3. Медицинская карта

1.	В блоке «Медицинская карта» указывается общая информация: номер карты, тип стационара, особый случай и вид оплаты.
{{a7a3a6ad-5b3e-4a25-8077-1326d895763c.png}}

h3. Персональные данные
 
2.	Блок «Персональные данные» заполняется персональными данными пациента. При создании МКСБ автоматически создается и сохраняется МКАБ по внесенным данным.
{{df7bda2b-5e52-41c5-a389-0cefaf5e27de.png}}

h3. Поступление

3.	В блоке «Поступление» указываются сведения о поступлении пациента на стационарное лечение. В зависимости от указания типа госпитализации (планово, экстренно или неотложно) зависит обязательный перечень заполняемости полей, если «планово» – заполняются все поля, если укажете «экстренно» - вышеуказанные поля уже являются необязательными к заполнению, если «неотложно» - обязательные поля к заполнению «Кем доставлен» и «Диагноз приёмного отделения»
{{42a73cd4-830a-4369-80b0-2f506aad4bbe.png}} 

При госпитализации пациента, устанавливается флажок «Госпитализация» и выбирается отделение, в которое направляется пациент для стационарного лечения.
Также в данном блоке можно добавить медицинскую запись в поле «Медицинские записи» нажмите «Добавить» и заполните шаблон «Первичный осмотр больного». (Для быстрого поиска шаблонов присутствует возможность добавления в избранные).
{{f2c7778b-e1a8-4f59-a649-e353308d2217.png}} 
Открывается шаблон «Первичный осмотр больного», заполняются только те поля, которые Вам необходимы. В печатной форме незаполненные поля не отображаются. 
Используя функцию «Словарь» можно сохранить слова, фразы и предложения для последующего выбора и отражения в поле документа.
После заполнения шаблона нажмите на кнопку «Просмотреть»
{{55d81e4c-4581-4ffd-b413-dcba5942167e.png}}  
Открывается окно «Предварительного просмотра» шаблона, для сохранения шаблона нажмите на кнопку «Все действия». Для вывода на печать, шаблон должен быть подписан.
{{91d3c563-a140-412a-9272-1e734d0385b4.png}}
 
Если пациенту отказано в госпитализации или он сам отказывается от неё, снимите флажок «Госпитализация». При этом автоматически будет включен флажок «Отказ от/в госпитализации».
Откроется поле для заполнения данных Причина отказа от/в госпитализации, Дата отказа, Время, Примечание к причине отказа от/в госпитализации.

{{4979a709-dd7b-4cb3-838c-edf34d221695.png}}

h3. Витальная информация

4.	В блоке «Витальная информация» указываются жизненно важные параметры. Данные заполнятся автоматически (при наличии). При необходимости добавить дополнительные данные пациента (например, рост, вес, давления или температуру) добавьте в раздел медицинские записи шаблон под названием «Дополнительная информация» и заполните те данные, которые Вам необходимы.

h3. Представители пациента

5.	В блоке «Представители пациента» указываются сведения о представителях пациента. Добавить представителя можно с помощью кнопки «+», расположенной под списком. Если новая карта пациента создается на основе найденной карты, то данные о представителях заполнятся автоматически (при наличии).

Чтобы сохранить созданный случай, нажмите на кнопку «Сохранить» затем нажмите на кнопку «Сохранить» после заполнения всех обязательных полей. 
Чтобы сохранить случай и осуществить переход в журнал пациентов, нажмите на кнопку «Сохранить» затем нажмите на кнопку «Сохранить и выйти».
{{e95c72a2-546b-4c35-974c-bbeb560ff626.png}} 
Для печати медицинской карты, статистической карты, журнала регистрации переливания, листа врачебных назначений по дате, листа назначений на дату, медицинской карты стационарного больного №003/у, температурный лист 1, лист наблюдений, лист назначений по часам, лист назначений АОКБ, нажмите на кнопку «Печать» и выберите необходимый вид карты для печати. 
{{644f2b9a-6e8d-4bd7-9f36-265de9d2555d.png}}

h2. Создание МКСБ на неизвестного пациента

Если в приемное отделение поступает пациент данные о котором неизвестный, заполняются только обязательные поля случайными данными.
Указываются произвольные данные ФИО, приблизительная дата рождения и в блоке поступление выбрать Экстренно , указав приемное отделение и отделение госпитализации. 

{{1b92127a-8be2-40a8-bfdc-88b0ce247987.png}}

Этих данных хватит чтобы сохранить карту. После того как о пациенте станет известно, данные нужно изменить.

h2. Печать информационных согласий и другой документации

h3.  Информационные согласия

При госпитализации пациента важно взять согласия. Согласия в МИС формируются двумя способами:
* Из раздела "Согласия"
* В разделе медицинские записи, с помощью медицинских записей.

Для добавления согласия и/или отказа через раздел "Согласия" перейдите в данный раздел, разверните блок, нажав на стрелку, затем нажмите кнопку "Добавить" 

{{3744986c-3bde-469d-8d32-7a5edca1443f.png}}

Карта пациента сохранится и произойдёт переход в "Добавление согласия/отказа пациента". В данном блоке заполните начала действия согласия (по умолчанию заполняется текущей датой). Если согласие и/или отказ будет действовать бессрочно, то обметьте это, либо выберете дату окончания действия согласия. При необходимости впишите дополнительную информацию. Специалист заполняется текущим пользователем, также его можно изменить

{{9e86055b-4abf-43f8-96d1-f122c3be4a6a.png}}

После добавления согласия нажмите "Сохранить", если всё заполнено корректно программа сообщит об успешном сохранении согласия

{{b8f5324e-dd1c-4d86-94d8-28d8e3414fe5.png}}

Для печати согласия/отказа нажмите на кнопку "Печать/ЭМД", откроется предварительный просмотр ПФ документа. Для отправки документа в РЭМД нажмите "Подписать и отправить"

{{2282a50e-1870-4553-bfbb-878178733b67.png}}

Для выхода из раздела "Добавление согласия/отказа пациента" нажмите "Вернуться к карте пациента" на боковой панели слева. 
После добавления согласия и/или отказа оно отобразится в карте пациента. Разверните данный раздел, нажав на стрелку, отобразятся все оформленные по пациенту документы. В данном разделе отображается:
* Дата начала действия согласия
* Дата окончания действия согласия
* Вид согласия
* Специалист, добавивший запись

Для редактирования записи нажмите на "Карандаш", произойдет переход в "Редактирование согласия/отказа пациента". Работа проводится аналогично добавлению новой записи.
Для удаления записи нажмите на "Корзину", запись будет удалена

{{884732c8-098e-4011-83f5-48b6ea280cf9.png}}

Для оформления ИДС через раздел "Медицинские записи" перейдите в одноименный раздел и нажмите "Добавить". Выберете мед запись, которая является ИДС

{{ecb233f5-7fd3-4528-b2ba-06ec1c8bbbf9.png}}

Заполните поля и нажмите кнопку "Просмотреть"

{{7ae92f4b-5872-440c-90c6-e09694e5ee6f.png}}

h3. Печать МКСБ

Печать карты по форме 003/у осуществляется по кнопке Печать/ЭМД , выбрать пункт ПФ 003/у МКСБ

h2. Нестандартные ситуации в приемном отделении

h3. Не сохраняется МКСБ

Если при сохранении МКСБ выходит ошибка "Проверьте правильность заполнения формы", внимательно обратите внимание на подсвеченные красным строки в МКСБ и дозаполните их.
{{0fd65b07-ae3a-43b0-8a6e-2085bad1d039.png}}

h3. Адрес: "Не определено" в ИДС/мед документах

Если в документах в адресе отображается "Не определено", в МКСБ нужно выставить формализованный адрес. 

{{4791eac5-6cfd-4be5-aa5c-c79ce8d4a0d7.png}}

h3.  Нет нужного адреса при ввода формализованного адреса 

Если при вводе адреса система не находит его, проверьте наличие адреса на сайте ФИАС 

h3.  Медицинская история отсутствует в БД

При возникновении данной ошибки, нужно почистить кеш браузера, после чего заново создать МКСБ.
